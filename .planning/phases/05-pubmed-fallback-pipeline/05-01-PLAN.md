<objective>
Create and export `searchPubMedForSupplement(name: string): Promise<RichPubMedArticle[]>` from `lib/services/pubmed-search.ts`, bridging the slim esummary article shape to the rich `medical-mcp-client.PubMedArticle` shape that `analyzeStudiesWithBedrock` requires. Add a unit test file covering the mapping and edge cases.

Purpose: `analyzeStudiesWithBedrock` in bedrock-analyzer.ts imports `PubMedArticle` from medical-mcp-client.ts (rich shape: pmid, abstract, authors as string[], journal, year, publicationTypes). The existing `executePubMedSearch` returns a slim shape (uid, no abstract, authors as {name:string}[], source). Passing the slim type directly to Bedrock causes runtime failures — pmid/abstract/authors undefined in the prompt builder. This plan resolves the type mismatch before any route wiring happens.

Output:
- `lib/services/pubmed-search.ts` with new exported function
- `lib/services/__tests__/pubmed-search-supplement.test.ts` with passing unit tests

      return articles.map(a => ({

        title: a.title,
